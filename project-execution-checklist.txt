Airbnb-Style Search Project - Execution Checklist

Setup
- Create repo structure: frontend/, backend/, root README.md, root .gitignore
- Scaffold frontend/ with React + TypeScript + Vite
- Install and configure Tailwind CSS
- Install and configure ESLint with Airbnb rules
- Add basic npm scripts: dev, build, lint, preview
- Scaffold backend/ with FastAPI app structure
- Add Python dependency management and run instructions
- Enable CORS for local frontend/backend development

Frontend Foundation
- Create folders:
  src/pages/search
  src/components
  src/context
  src/services
  src/types
  src/mocks
- Add global Tailwind theme tokens for colors, spacing, radius, shadows
- Define TypeScript types for:
  ListingItem
  SearchParams
  SearchResponse
- Set up API client wrapper in src/services

Search Page UI
- Build src/pages/search/index.tsx as the main page entry
- Create page shell inspired by Airbnb layout
- Build top search bar
- Build filter controls area
- Build results section container
- Add responsive behavior for mobile/tablet/desktop
- Decide whether right-side map-style panel is:
  optional placeholder for visual balance
  or skipped for MVP

Reusable Components
- SearchInput
- FilterBar
- ListingCard
- ResultsList
- LoadingState
- EmptyState
- ErrorState
- LoadMoreButton or pagination controls

Listing Card Requirements
- Show item image
- Show item title
- Show item price
- Show external listing link
- Add hover/focus styling
- Keep layout clean for long titles and broken images

State Management
- Start with local page state in src/pages/search
- Introduce React Context only if search/results state is needed across distant components
- Avoid unnecessary prop drilling, but also avoid context for trivial flows

Backend Foundation
- Create FastAPI app entrypoint
- Add route module for search
- Add DTO/schema module using Pydantic
- Add service layer for filtering/search logic
- Add mock data module with structured item objects

Backend DTOs / Models
- ListingItemDto
- SearchQueryDto
- SearchResponseDto
- Optional enum/model for filter values if we define allowed filters

Backend API
- Implement GET /search
- Support query params:
  search
  filter
  take
  offset
- Return structured response:
  items
  total
  take
  offset

Backend Validation
- Validate search as optional trimmed string
- Validate filter against allowed values
- Validate take as integer with min/max bounds
- Validate offset as integer with minimum 0
- Return proper validation errors for bad input

Mock Data
- Create realistic eBay-style dummy listing objects
- Include enough data for searching/filtering/pagination
- Keep data outside route handlers
- Ensure each item includes:
  title
  price
  image
  link

Frontend Integration
- Wire search page to backend /search
- Send search, filter, take, offset
- Render backend results into listing cards
- Show loading state during fetch
- Show empty state when no results
- Show error state on failed request
- Handle external links safely

Interaction / UX
- Decide search trigger:
  button click only
  or debounced typing
- Add filter change behavior
- Add pagination or Load more
- Preserve search input and current results state during interaction
- Make UI keyboard accessible

Polish
- Match Airbnb-inspired visual quality with Tailwind utilities/components
- Improve spacing, shadows, card hierarchy, and typography
- Verify responsive layout at common breakpoints
- Add fallback image handling
- Refine empty/error copy

Quality
- Run frontend lint and fix issues
- Add backend formatting/linting if time allows
- Add at least one backend test for /search
- Add at least one frontend test for results rendering or empty state
- Smoke test the full local flow

Submission
- Write README with setup/run steps
- Document tech stack choices
- Note that backend currently uses dummy data
- Mention supported query params and validation
- Final pass for code cleanup and naming consistency

Suggested Milestones
1. Scaffold everything and get both apps running
2. Finish backend /search with DTOs and validation
3. Build search page UI with mock data
4. Connect frontend to backend
5. Add polish, lint, README, and testing

Best MVP Cut If Time Gets Tight
- Keep one search bar
- Keep one optional filter
- Use Load more instead of full pagination
- Skip map panel
- Skip advanced context unless clearly needed
